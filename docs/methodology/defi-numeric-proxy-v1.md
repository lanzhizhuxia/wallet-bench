# DeFi Numeric Proxy v1.1

**Date**: 2026-03-06
**Version**: v1.1
**Status**: Informative
**Purpose**: 仅用于可视化和敏感性分析，不作为采购决策依据

---

## 1. 目的声明

本文档定义了 DeFi 场景 Agent 集成难度评级到数值分数的映射规则。

**⚠️ 重要声明**：
- 数值代理（Numeric Proxy）**仅用于** dashboard 可视化和权重敏感性分析
- **不可**单独作为采购结论或最终排名依据
- 定性评级始终是主要数据
- 任何基于数值代理的排名结论**必须**附带本声明

---

## 2. 映射规则

评级从 AI Agent 集成视角出发，关注 Agent 自主完成 DeFi 场景的难度，而非人类开发者的工时估算。

| 定性评级 | 中文 | 数值代理 | 含义 |
|----------|------|---------|------|
| 🟢 ready_to_use | 即用 | **1.00** | SDK/工具内置该场景，Agent 声明即用，无需额外编码 |
| 🟢 ready_to_use† | 即用† | **0.80** | 内置能力可用但有限制条件（如仅聚合器、仅特定链） |
| 🟡 low_barrier | 低门槛 | **0.65** | 需要 calldata 编码但步骤少，有 ABI 即可完成 |
| 🟠 moderate | 中等 | **0.40** | 多步交易 + 协议专属编码 + 外部依赖或需人工介入 |
| 🔴 not_feasible | 不可行 | **0.00** | 架构缺少必要能力（签名、calldata、密钥导出等） |

### 评估维度

| 维度 | 说明 |
|------|------|
| **集成模式** | 声明即用 / calldata 组装 / 协议专属编码 / 不可行 |
| **交易步数** | 完成该场景需要几步链上交易（0 = 内置 action） |
| **外部依赖** | 是否需要协议 ABI、第三方 API、自定义编码器 |
| **人工介入** | Agent 能否全自主完成，还是需要人工确认 |

## 3. 场景权重策略

### 默认: 等权
每个场景权重 25%。适用于"不知道业务侧重"的初始评估。

### 可选权重集

| 权重集 | Uniswap | Aave/Morpho | Hyperliquid | Polymarket | 适用场景 |
|--------|---------|-------------|-------------|------------|----------|
| **DeFi-Heavy** | 30% | 30% | 20% | 20% | 主要做链上 DeFi |
| **Trading-Heavy** | 20% | 10% | 40% | 30% | 主要做交易/投机 |
| **等权 (Default)** | 25% | 25% | 25% | 25% | 通用评估 |

---

## 4. 计算公式

```
DeFi Score = Σ (scenario_proxy × scenario_weight) × 100
```

### 示例: Privy (等权)

| 场景 | 评级 | 代理值 | 权重 | 加权值 |
|------|------|--------|------|--------|
| Uniswap | 低门槛 | 0.65 | 25% | 0.1625 |
| Aave/Morpho | 低门槛 | 0.65 | 25% | 0.1625 |
| Hyperliquid | 低门槛 | 0.65 | 25% | 0.1625 |
| Polymarket | 中等 | 0.40 | 25% | 0.1000 |
| **Total** | | | | **0.5875** |

**Privy DeFi Score (等权) = 58.8%**

## 6. 禁用场景

以下场景下**禁止**使用本数值代理：
1. 作为唯一排名依据向 leadership 汇报
2. 与技术跑分简单相加生成"综合分"（量纲不同）
3. 不附带免责声明的任何展示
4. 权重未经 PM 确认的正式采购建议
